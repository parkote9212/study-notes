### **Q66. 프로젝트에서 가장 어려웠던 점**

**면접관**: BizSync 프로젝트에서 가장 어려웠던 기술적 문제는?

**답변**:

```
WebSocket 연결이 끊겼다 재연결될 때
중간에 발생한 업데이트를 놓치는 문제였습니다.

상황을 설명드리면,
사용자 A가 칸반 보드를 보고 있는데
네트워크가 잠깐 불안정해서
WebSocket이 끊어졌다가 다시 연결됐습니다.

그 짧은 시간 동안 다른 사용자가
Task를 생성하거나 이동시켰는데,
사용자 A는 그 업데이트를 못 받는 거죠.

처음에는 재연결될 때마다
전체 보드를 다시 불러오게 했는데,
깜빡임이 심하고 사용자 경험이 안 좋았습니다.

해결 방법은 이렇습니다.

마지막 동기화 시간을 저장해두고,
재연결 시 그 이후의 변경사항만
서버에 요청하는 방식으로 바꿨습니다.

const lastSync = useRef(Date.now());

onConnect: () => {
  // 놓친 업데이트 가져오기
  fetchUpdates(lastSync.current).then(updates => {
    updates.forEach(applyUpdate);
  });
  lastSync.current = Date.now();
}

백엔드에도 API를 추가했습니다.

@GetMapping("/api/projects/{id}/updates")
public List<Update> getUpdates(
  @PathVariable Long id,
  @RequestParam Long since
) {
  return updateService.findUpdatesSince(id, since);
}

이렇게 하니까 재연결 시
놓친 부분만 조용히 반영되고
사용자는 끊김을 거의 못 느낍니다.

이 문제를 해결하면서
분산 시스템에서 네트워크는 항상 불안정하고,
클라이언트는 재연결을 항상 고려해야 한다는 걸
배웠습니다.
```
